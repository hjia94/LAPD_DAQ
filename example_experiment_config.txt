# Example experiment_config.txt for LAPD_DAQ
#
# Copy this file to experiment_config.txt and edit IP addresses before running.
# The active defaults are intentionally small so the hardware diagnostic checks
# can be run on the instrument PC without starting a full scan.
#
# Recommended first checks (edit flags at the top of the test file, then run):
#
#   Real scope, no motor movement:
#     Set RUN_DATA_RUN_SCOPE_CHECK = True and EXPERIMENT_CONFIG_PATH in
#     tests/test_hardware_instruments.py, then:
#     pytest tests/test_hardware_instruments.py::DataRunScopeHardware -v -s
#
#   Real motor movement, fake delayed scope data:
#     Set RUN_DATA_RUN_MOTION_CHECK = True and MOTION_ALLOW_MOVE = True in
#     tests/test_hardware_instruments.py, then:
#     pytest tests/test_hardware_instruments.py::DataRunMotionHardware -v -s
#
# WARNING: the motion check WILL move the configured motors when
# MOTION_ALLOW_MOVE is True. Verify that the active [position] target below
# is safe for the installed probe and current LAPD configuration first.
